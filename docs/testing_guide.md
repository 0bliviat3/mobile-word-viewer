# 테스트 가이드: Mobile Word Viewer

## 1. 테스트 개요
이 문서는 Mobile Word Viewer 애플리케이션의 핵심 기능들을 테스트하는 가이드입니다. 애플리케이션은 .docx 파일을 오프라인에서 렌더링하는 React 기반 모바일 어플리케이션으로, React 18+, Vite, mammoth.js, JSZip, Tailwind CSS로 구성되었습니다.

## 2. 테스트 환경 준비
### 2.1 시스템 요구사항
- Node.js 16 이상
- npm 8 이상
- Git
- Android Studio (Android 테스트용) 또는 Xcode (iOS 테스트용)

### 2.2 의존성 설치
```bash
npm install
npm install mammoth.js jszip
```

## 3. 핵심 기능 테스트
### 3.1 파일 선택 및 로드 테스트
#### 3.1.1 일반 문서 파일
- `.docx` 확장자 파일을 선택하여 로드
- 파일 형식이 아닌 다른 파일은 rejection 확인
- 50MB 이상의 큰 파일의 경우 오류 처리 확인

#### 3.1.2 드래그 앤 드롭
- 파일을 브라우저 창으로 드래그 앤 드롭
- 올바른 .docx 파일이 아닌 경우 오류 메시지 확인

### 3.2 문서 렌더링 테스트
#### 3.2.1 텍스트 렌더링
- 다양한 폰트 크기 및 스타일이 포함된 문서 확인
- 제목, 본문, 목록이 올바르게 렌더링되는지 확인

#### 3.2.2 표 렌더링
- 문서 내 포함된 표가 올바르게 표시되는지 확인
- 가로 스크롤이 작동하는지 확인

#### 3.2.3 이미지 렌더링
- 문서 내 포함된 이미지가 올바르게 표시되는지 확인
- 이미지가 확장 가능한지 확인

### 3.3 기능 테스트
#### 3.3.1 줌 기능
- 버튼을 통한 줌 인/아웃 기능 확인
- 핀치 제스처를 통한 줌 기능 확인
- 더블 탭으로 100% 또는 적합한 너비로 토글 확인

#### 3.3.2 반응형 레이아웃
- 세로/가로 모드에서 올바르게 표시되는지 확인
- 안드로이드/IOS 기기에서 동작 확인

### 3.4 에러 처리 테스트
#### 3.4.1 잘못된 파일 형식
- .txt 또는 다른 형식의 파일이 업로드될 경우 적절한 오류 메시지 확인
- 빈 문서나 손상된 문서 처리 확인

#### 3.4.2 네트워크 관련 오류
- 오프라인 환경에서 작동 확인
- 인터넷 연결이 없는 상태에서도 문서 로드 가능 확인

## 4. 성능 테스트
### 4.1 로딩 시간 체크
- 1MB 문서: 3초 이내 로딩
- 10MB 문서: 10초 이내 로딩 (로딩 인디케이터 표시)

### 4.2 메모리 사용량
- 대용량 문서(100페이지 이상) 처리 시 메모리 누수 없음 확인
- 파일 교체 반복 시 메모리 증가 없음 확인

## 5. 브라우저 호환성 테스트
- Chrome 모바일 (Android)
- Safari 모바일 (iOS)
- Samsung Internet
- Android WebView (Capacitor)
- iOS WKWebView (Capacitor)

### 6.1 기능 정상 작동 확인
- 문서 로드 및 렌더링 정상 작동
- 줌 기능 정상 작동
- 반응형 레이아웃 정상 작동

### 6.2 에러 상황 처리
- 잘못된 파일 형식에 대해 적절한 오류 메시지 표시
- 빈 문서에 대해 안내 메시지 표시

### 6.3 성능 기준 충족
- 문서 로딩 시간 기준 충족
